# eNSP（华为模拟器）

# 链路聚合

# manual

优点

  • 手工链路聚合，成员接口手工配置加入，没有 LACP 的参与。
  • 当需要在两个直连设备间提供一个较大的链路带宽而设备又不支持 LACP 协议时，可以使用手工模式。
  • 不同厂家之间无需担心协议报文协商问题。

缺点

  • 当一条链路故障时，故障链路无法转发数据，链路聚合组自动在剩余的两条活动链路中分担流量。
    手工模式 Eth-Trunk 可以完成多个物理接口聚合成一个 Eth-Trunk 口来提高带宽，同时能够检测到同一聚合组内的成员链路有断路等有限故障，但是无法检测到链路层故障、链路错连等故障。
    • 单根纤芯发生故障时，可能出现收端正常的一方端口处于 UP，而出现单通，所以这种情况一般要求端口匹配状态为自适应状态。

# LACP 模式链路聚合

优点

  • 提高 Eth-Trunk 的容错性，并且能提供备份功能，保证成员链路的高可靠性

缺点

  • 不同厂家对接可能因为协议报文的处理机制等不同，产生对接异常。
  • 而且一般来说：动态聚合要同端口匹配方式为强制相配套使用，因为：如果端口匹配方式为自适应，那么当物理链路质量不好时，可能端口状态频繁出现变化，相应的聚合组状态也会频繁出现 up、down 故障。

不同厂家对接，建议设置聚合组为静态、端口设置为自适应方式

# max active-linknumber

  • 一般来说聚合的线路为三条就是选择 max active-linknumber 2，这样即时有一条 down 了也可以即时补进去一条。还是为 2 条在工作

# stp edged-port enable & disable

  • stp edged-port enable 配置的接口不参与生成树计算，即可以转发 BPDU 也可以接收 BPDU

  • :stp disable 禁用 STP/RSTP/MSTP 功能，如果在链接客户端的接口上配置了此命令，这个接口禁用 STP/RSTP/MSTP 功能，即不参与生成树计算也不转发 BPDU 报文

  • 定义：在 RSTP 里面，如果某一个指定端口位于整个网络的边缘，即不再与其他交换设备连接，而是直接与终端设备直连，这种端口叫做边缘端口。

  • 特点

  • 边缘端口的特性是端口 up 后可直接进入 forwarding 状态，而不需要等待转发延时，所以边缘端口连接的设备是终端设备，电脑，服务器等需要快速访问网络的设备。
    • 场景 1：在连接服务器，DHCP 客户端，防火墙，打印机，路由器等终端设备可以使用边缘端口
    • 场景 2：重要服务器，避免在网络发生拓扑变化时被阻塞，同时可以使其快速上线
    • 场景 3：可以用于连接 DHCP 客户端，使其客户端快速获得 IP 地址
    • 场景 4：在大型网络中，连接大量终端 PC 的交换机可以使用边缘端口，因为当 PC 开机时，连接 PC 的端口为 DP 会 UP 并进入转发状态，交换机会产生大量的 TC BPDU，导致网络中的交换机频繁刷新自身的 MAC 地址表。

一般来说开启 stp enable 为 pc 机接口上，应为它不需要参与生成树计算

# Stp cost 200000

  • stp cost 是生成树协议在选举根端口和指定端口时需要使用的一个重要参数。开销越小被选举为根端口或指定端口的可能性就越大。stp cost 100000 这么大的开销值，这个端口基本上就是出于逻辑阻塞状态了，除非交换机上其他端口上的物理线路都断开了，这个才有可能进入到转发状态。
  • 说人话就是设置了这个他就是堵塞的，应为它的优先级很低，除非正常线路断开否则它不会启用。

# stp bpdu-protection

  • 需要配置在 BR 根端口，使用命令 display stp brief 查看 Root 状态是哪个，哪个就是根端口。

root = 根端口 alte = 替代端口 = alternative desi = 指定端口 = designation
